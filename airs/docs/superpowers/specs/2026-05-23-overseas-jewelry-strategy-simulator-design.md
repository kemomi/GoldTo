# 海外珠宝市场战略模拟器 Agent 设计稿

## 1. 设计结论

本项目第一版产品定位为：

`一个面向海外市场策略分析师的对话式战略模拟器 Agent。`

它在每日晨会前自动整合公开市场情报，筛选高优先级竞品与合规事件，生成结构化战略简报；当分析师点选 `Top 1 / Top 2` 竞品事件后，系统进一步比较不同价格/促销应对级别的潜在影响，并输出带证据、带置信度、可人工接管的推荐结论。

本设计优先满足以下目标：

- 解决公开情报搜集慢、判断标准乱、决策反应晚的问题
- 贴合评审重点，突出业务落地、风险安全、演示表达和非技术可理解度
- 在不接入企业内部数据的前提下，完成从情报到建议再到策略模拟的完整闭环
- 将 MVP 范围控制在 `1 天可交互跑通` 的复杂度内

## 2. 业务背景与核心场景

### 2.1 企业映射

企业 X 映射为一家以珠宝饰品为主的出海品牌企业，主营：

- 黄金首饰
- 钻石彩宝镶嵌饰品
- 婚嫁轻奢珠宝
- 工艺典藏首饰

销售渠道以 `线下实体门店分销` 为主，辅以 `跨境电商平台` 与 `品牌官方独立站`。

### 2.2 单一打穿场景

场景标准范式：

`谁 + 在什么时间 + 基于哪些公开信息 + 做什么核心决策 + 当前为什么慢/乱/晚`

具体定义如下：

- 谁：海外市场策略分析师
- 时间：每日晨会前固定时段
- 信息来源：公开舆情、竞品动态、跨境电商平台官方公告、海外社交媒体趋势、当地行业新闻、跨境合规政策更新、公开行业报告
- 决策动作：快速筛选多市场冗余信息，统一标准研判风险与增长机会，输出标准化每日市场战略简报，支撑总部、区域运营与市场团队当日决策
- 当前问题：
  - 慢：人工搜集多国多渠道公开信息，耗时 2-3 小时
  - 乱：不同分析师判断标准不一致，成果不可比、不可沉淀
  - 晚：对调价、促销、合规、舆情等变化反应滞后 1-3 天
  - 缺：缺少可直接转成行动的晨会输入

## 3. 用户与使用方式

### 3.1 核心用户

- 唯一主角色：海外市场策略分析师

### 3.2 间接受益角色

- 总部管理层：接收战略微调建议
- 区域运营负责人：接收竞品应对、巡店与活动应对建议
- 市场策略岗位：接收趋势观察与内容方向建议
- 合规/运营专员：接收合规排查优先级建议

### 3.3 使用方式

- 产品形态：`对话式 Agent`
- 第一眼观感：`智能分析师感`
- 日常流程：
  1. 系统每日自动生成战略简报
  2. 分析师阅读简报并选中 `Top 1 / Top 2` 事件
  3. 用户手动触发“开始模拟”
  4. 系统对最高优先级事件输出多方案对比与推荐结论

## 4. 市场、语言与信息边界

### 4.1 第一版重点市场

按优先级排序：

1. 中国香港
2. 新加坡
3. 美国
4. 加拿大
5. 泰国

### 4.2 第一版语言支持

- 必须支持：简体中文、繁体中文、英文
- 可后置：泰文及其他语种

处理原则：

- 保留原文
- 提供中文摘要

## 5. 核心业务目标与能力优先级

### 5.1 当日必须推动的 3 类业务动作

1. 竞品竞争预警
   区域运营负责人当天对标核心商圈竞品门店活动、定价、上新动作，调整巡店重点与临时促销应对预案
2. 跨境合规风控处置
   合规与区域运营专员当天核查贵金属标识、钻石溯源、广告宣传等新规风险，排查门店物料与线上内容
3. 区域趋势机会跟进
   海外策略分析师当天更新婚嫁、轻奢、可持续珠宝趋势，同步新品观察清单与社媒内容方向

### 5.2 能力优先级

1. 第一优先：竞品竞争预警
2. 第二优先：跨境合规风控
3. 第三优先：区域趋势机会

第一版北极星能力聚焦 `竞品竞争预警`。

## 6. 重点竞品与监测事件

### 6.1 重点竞品

- 周生生、六福珠宝
- 谢瑞麟
- Signet 集团
- 卡地亚、蒂芙尼

### 6.2 重点事件类型

- 新店开业 / 闭店
- 商圈门店迁移
- 价格调整
- 金价联动策略
- 婚嫁系列上新
- 轻奢系列上新
- 节日促销活动
- 联名合作
- KOL 投放
- 百货渠道入驻或合作变化
- 品牌负面舆情

## 7. 情报筛选规则

系统对竞品竞争预警采用三级筛选：

价格异动阈值说明：

- `大于等于 5%`、`2%-5%`、`小于 2%` 作为第一版默认判定阈值
- 该阈值在实现时应设计为 `可动态配置参数`
- 开发时预留简单调整入口，便于不同市场口径适配与演示调试

### 7.1 必须上报

- 来源：品牌官方公告、商场官方公示、权威行业媒体
- 相关性：核心竞品、重点市场、核心商圈、黄金/婚嫁/轻奢主力品类
- 影响：新开/关停/迁店、调价幅度大于等于 5%、大型促销、重磅上新、高端联名代言、主流渠道合作变动
- 证据：原始链接或截图，至少 1 个权威信源，信息完整
- 时效：24 小时内

### 7.2 可选上报

- 来源：本地知名生活媒体、行业头部 KOL
- 相关性：核心竞品、重点市场内的衍生品类或局部活动
- 影响：调价 2%-5%、单商圈小型促销、常规款上新、小范围 KOL 推广
- 证据：单条有效信源，关键信息清晰
- 时效：24-48 小时

### 7.3 直接忽略

- 来源：普通个人账号、匿名论坛、无资质自媒体
- 相关性：非核心竞品、非重点市场、无关小众品类
- 影响：调价小于 2%、琐碎运营动作、无明显市场冲击
- 证据：缺失、冲突或无法追溯
- 时效：超过 48 小时且无跟进价值

### 7.4 同级高风险事件排序规则

- 遵循 `不漏报高风险事件` 的底线
- 当同级事件数量超出简报容量时，优先收录 `核心商圈` 相关异动
- 在品类层面，优先收录 `黄金` 与 `婚嫁` 主力经营品类相关异动
- 若仍需继续排序，则优先保留对当日门店客流、成交与价格体系冲击更直接的事件

## 8. 每日战略简报结构

简报要求 `3-5 分钟读完`，固定包含：

1. `全域态势速览`
   1 段，概括 5 大市场整体竞争烈度与风险等级
2. `高优竞品异动`
   最多 5 条，单市场最多 1 条，仅收录必须上报级竞品事件
3. `合规红线提醒`
   最多 2 条
4. `市场机会看点`
   最多 2 条
5. `分角色落地行动`
   总部管理层 1 条、区域运营 1 条、市场策略岗 1 条
6. `情报溯源索引`
   每条事件附来源链接或凭证标注

## 9. 信息源策略

### 9.1 第一版必须真实接入

- 竞品官方源
  - 品牌官网
  - 新闻稿
  - 门店公告
  - 官方社媒账号中的公告型内容
- 商场 / 渠道官方源
  - 商场官网
  - 百货公告
  - 电商平台规则页

### 9.2 可后置或预置导入

- 行业 / 媒体源
- 社媒 / KOL 源

信息源策略原则：

- 以 `分析决策链完整性` 为第一优先
- 以 `真实抓取的核心一手信源` 证明系统可用性
- 以 `预置样本` 补齐业务场景广度

## 10. 战略模拟器定位

第一版不是纯情报简报系统，而是：

`简报 + 策略建议 + 影响模拟型 Agent`

更希望评委记住它：

`会帮人做选择`

### 10.1 模拟主方向

- 主方向：价格 / 促销应对

### 10.2 模拟输出形式

- 输出多个动作方案对比
- 输出各自影响
- 不输出具体价格数字

### 10.3 固定应对级别

- 保持不动
- 局部跟进
- 限时跟进

`升级关注` 作为证据不足、风险过高或需上收决策时的降级结论，不作为常规对比方案。

## 11. 影响模拟逻辑

### 11.1 证据体系

采用 `混合型`：

- 规则评分为主
- 公开信号做校正

### 11.2 四个核心评估维度

按优先级排序：

1. 毛利 / 价格体系压力
2. 客流分流风险
3. 品牌调性受损风险
4. 合规风险

### 11.3 模拟触发方式

- 系统先自动出简报
- 分析师对 `Top 1 / Top 2` 事件手动点击“开始模拟”
- 第一版只深度模拟 `1 个最高优先级事件`

### 11.4 模拟详情固定模块

1. 事件核心概况
2. 三类可选应对级别方案
3. 四维影响横向对比
4. 优选方案及判定理由
5. 后续观测与执行提示

## 12. 风险安全机制

第一版明确包含以下机制：

1. 证据门槛
   缺少原始链接或权威来源的高风险事件，不得进入核心简报
2. 低置信度隔离
   高相关但证据不完整的内容进入 `待人工复核区`
3. 降级输出
   证据不足、信息冲突或关键字段缺失时，只输出“升级关注”或“建议人工确认”
4. 全链路可追溯
   每条结论都可回看来源链接、原文摘录、中文摘要、抽取结果与评分理由
5. 人工接管
   分析师可调整优先级、重新触发模拟、标记误报、决定是否采用建议

底线倾向：

- 最不能接受 `漏报高风险事件`

高相关但证据不完整的事件处理策略：

- 进入 `待人工复核区`
- 明确标注低置信度
- 不直接给动作建议

## 13. 产品交互设计

### 13.1 产品形态

采用 `对话式 Agent + 结构化结果卡片`

### 13.2 主界面结构

- 左侧：今日简报目录区
  - 五大市场
  - Top 事件
  - 待复核事件
  - 合规提醒
- 中间：Agent 对话主区
  - 回答追问
  - 解释判定依据
  - 引导进入模拟
- 右侧：证据与模拟详情区
  - 来源链接
  - 原文摘要
  - 方案对比卡片
  - 推荐结论

### 13.3 核心交互流程

1. 进入系统即展示今日战略简报
2. 用户追问今日最高风险事件及判定依据
3. 用户点击“开始模拟应对方案”
4. 系统展示三方案与四维影响对比，并给出推荐结论与执行提示

## 14. 对话能力边界

比赛现场重点回答以下 3 个问题：

1. 今日五大重点市场里，哪起竞品调价促销事件风险等级最高，判定依据是什么
2. 面对此次竞品促销攻势，不同应对策略分别会产生哪些利弊影响
3. 综合多方因素最优选择哪种应对级别，各岗位后续该落实哪些相关工作

Agent 对话能力只围绕：

- 事件解释
- 证据追溯
- 方案对比
- 推荐理由
- 后续动作

不扩展为开放式百科问答。

## 15. 技术架构

### 15.1 实现原则

- 架构必须能在 `1 天内完成可现场跑通原型`
- 保留完整业务闭环
- 不为追求 Agent 概念而堆叠复杂自治框架

### 15.2 推荐技术栈

- 前端：Next.js 或 React + Vite
- 后端：Python + FastAPI
- 编排：LangGraph 或自定义 pipeline
- 存储：SQLite
- 模型：OpenAI API 或同类大模型
- 检索：轻量全文检索，向量库非必需

### 15.3 核心流程节点

1. Source Ingest
2. Normalize
3. Event Extract
4. Priority Score
5. Brief Generate
6. Scenario Simulate

## 16. MVP 范围与非目标

### 16.1 MVP 必做

- 真实接入部分官方源
- 支持简中 / 繁中 / 英文
- 自动生成每日简报
- 自动识别并呈现 Top 事件
- 手动触发 Top 事件模拟
- 输出三类应对方案
- 输出四维对比
- 输出推荐理由与岗位动作
- 提供来源链接、置信度、待人工复核机制

### 16.2 明确不做

- 全市场全量实时采集
- 对所有事件都深度模拟
- 具体价格数字推荐
- 重型知识库与多 Agent 自治系统
- 长周期趋势预测模型

## 17. 演示方式

比赛演示采用 `混合式`：

- 真实抓取 1-2 类核心源
- 其余使用预置数据补齐场景

目标侧重：

- 让评委感受到 `业务闭环最完整`
- 同时保留 `真实抓取能力` 作为系统可用性佐证

演示呈现原则：

- 演示环节优先展示 `真实爬虫抓取生成` 的事件案例
- 预置数据主要用于补足市场、品类与场景广度，不作为首个展示重点
- 路演叙事中应明确区分“真实抓取结果”与“场景补足样本”，强化系统真实可用观感

### 17.1 推荐演示顺序

1. 展示今日战略简报，优先点开由真实抓取生成的高优事件
2. 追问最高风险事件与判定依据
3. 进入模拟
4. 展示三方案四维对比
5. 展示推荐结论、岗位动作与证据追溯

## 18. 评审维度对齐

- 业务落地潜力：晨会高频场景、真实角色、明确动作闭环
- 技术实现：真实抓取、结构化抽取、规则评分、模拟逻辑、可运行原型
- 风险安全：证据门槛、复核区、降级输出、人工接管、可追溯
- 创新性：从情报聚合升级到策略比较
- 演示表达：简报到模拟的闭环清晰
- 非技术可理解度：统一使用业务语言表达结果

## 19. 实施约束说明

默认执行约束按以下条件设计：

- 开发时间：1 天
- 团队：1 人
- 目标：必须完成可现场交互跑通的原型与 README
- 技术复杂度：中等
- 允许使用现成大模型 API 与工作流框架

补充说明：

本设计虽然按 `1 人 / 1 天` 收敛范围，但模块边界清晰，也支持在多人协作时按前端、后端、规则层、集成层并行拆分开发。

## 20. 成功标准

如果第一版完成以下结果，即视为设计达标：

- 能在首页稳定展示一份当日战略简报
- 能清楚说明为什么某条事件是当日最高优先级
- 能对 `Top 1` 事件完成三方案对比与四维评估
- 能给出可解释的推荐结论，而不是泛泛建议
- 能展示来源、置信度、复核与人工接管机制
- 能让非技术评委在 1 次演示中看懂“它如何帮助人做选择”
